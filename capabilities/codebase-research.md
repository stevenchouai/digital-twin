# 代码研究

对开源项目、工具仓库或重构中的代码库进行深度分析，提取架构洞察与可迁移方法。

## 目标

- 快速建立心智模型
- 找到用户价值与边界
- 判断哪些能力值得迁入 Steven 的工作流

## 五阶段流程

### Phase 1：定向

- 读 README、package manifest、LICENSE
- 映射顶层结构与入口文件
- 识别项目类型与技术栈

### Phase 2：架构深潜

按优先级调查：

- 入口流程
- 核心循环
- 数据流
- 扩展模型
- 状态管理
- 并发模型
- 安全边界

### Phase 3：用户价值

回答：

- 用户具体能做什么
- 交互模型是什么
- 相比替代方案独特在哪
- 明确不能做什么

### Phase 4：迁移机会

分三层判断：

- 可直接借鉴的方法
- 可提取复用的模块
- 值得新建的工具或流程

### Phase 5：结构化报告

输出：

- TL;DR
- 架构概览
- 用户价值
- 可迁移机会
- 推荐下一步

## 数字分身视角

如果研究的目的与 Steven 的个人系统有关，额外回答：

- 它能增强哪个已有能力模块
- 它补的是流程、runtime，还是仅仅是文案包装
- 应不应该写入 learning note
